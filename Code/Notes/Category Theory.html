I've been watching category theory lectures on YouTube lately, by two people: Richard Southwell, and Bartosz Milewski. The following are notes I've synthesized from these lectures.

In Milewski's video, he first talks about the nature of mathematics and programming, as it is taught to young people. He claims that the current state of mathematics education regards young people too much as computers, and as such, teaches them to carry out instructions without mistake, as we expect computers to do. This is a poor way to teach humans, who are far better at thinking globally, abstractly, and goal-orientedly than locally, technically, and routinely. He continues to relate this dichotomy to that of functional vs. imperative programming languages, saying that the former is more condusive to human thinking, and the latter mechanical thinking. Category theory then is the functional programming approach to mathematics, one might say. It allows people to exchange abstract ideas in the way least burdened by technicalities and specifics.

As for what category theory is... we can certainly start with how a category is defined. Like functional programming, like lambda calculus, this branch of mathematics is very simply defined, the complexity arising out of the creative implementations and emergent properties of the simple definition. This makes starting over talking about category theory funny, because I will repeat myself not feeling like I know much more about these simple rules, because I already grapsed the rules themsevles the first time I'd heard them. Oh, it is a graph, with nodes and edges. But now my understanding has been upbuilt from this definition, and so in beginning again, I will build deeper and taller out of the wood and nails that have been the same as in every other beginning.

A category is some collection of objects (objects) and arrows between them (morphisms), given that this collection respects two properties: a) that arrows are associative, in that a arrow from a to b and a arrow from b to c can combine to a arrow from a to c, and b) that each object has an identity arrow, that takes itself to itself. Thus, the category is defined, and category theory defined thereby as its study. The category can represent anything, from objects as numbers and arrows as functions between them, to objects as words and arrows as concatenation into sentences.

Milewski then goes on to define the category of types that exist, when considering the theory of programming, which I will attempt to relate here.

To set the stage, the categeory we are considering is one where objects are data types, and arrows are functions that would take as input some type, and output some other type. More specifically, an outgoing arrow is called an Elimination, due to the function eliminating the type that it came from, transforming to a different type, and an incoming arrow is called a Constructor, due to the function constructing the type from another.

Now, we can define the empty type, 'Void'. This type corresponds to falsehood in logic, or the empty set in set theory. Void type in category theory is a object with arrows to every other type imagineable in the category– it is the Initial Object of the category. It can eliminate to any other type.

Moving on, we can define the 'Unit' type. In set theory, this is a singleton set. This is the opposite of the Initial Object, which is called the Terminal Object. As an inversion of the former, the Terminal Object has a unique arrow from any object to it. As such, the Unit type can be constructed from any other type.

Now that we have some types, we can begin to explore the operations we can perform over them, given that they are objects in category theory. The first of these we'll consider is the Product. As defined, the product of two objects A and B is some object C that has arrows to both A and B. Given that there could exist many of these objects in any category, the 'Candidate' language is employed. The product operation, as spoken about by both Milewski and Southwell, is always spoken about as having 'Candidates', which is such funnily consistent language. How do we know which candidate is the best, and ultimately is the product? Well, the best product is the one which has some incoming arrow h that takes any other product candidate to it. That way, the actual product, the best product, is the most direct product of the two. Like how you could ask what is the product of 3 and 4, and while there are infinite numbers that are products of 3 and 4 (and whatever else), only one number is the most direct product of 3 and 4, which is 12. Any other candidate product could be reduced to 12 multiplied by whatever else.

As is pattern in category theory, one thing we can do is flip the polarity of all arrows in a construction, and get another construction for free, as we did between Initial and Terminal Objects. So, by flipping all arrows in our definition of a product, we get the notion of co-product, or Sum. Now, we have a construction where the co-product candidate is one to which two objects A and B point, this object being thought of as a disjoint union of A and B– either A or B, not both. The best co-product would then be the one which would have an arrow h to any other candidate, which is to say that the best candidate is the one that is closest constructed from A and B.

From these two operations, we have tools to discuss Monoidal Categories. As Milewski says, a monoidal categeory is one in which every two objects have a product and a coproduct. Furthermore, the product and coproduct operations need to have associativity and an identity. Associativity is respected by both given the lack of respect for order in their definitions, and the identities are given by the properties that the product between some type and the unit type is itself, and that the coproduct between some type and the void type is itself.

And badda-bing badda-boom, we can now construct any data type from these rules. Boolean is the coproduct of two units, thereby giving a bool the choice of being either the first unit or the second, where one is arbitrarily given as true and the other as false. Natural numbers can be represented as strings of coproducts, as many as is the magnitude of the number to be represented.

There is a definition of categories that doesn't use objects at all, only arrows. A category is a collection of arrows with two idempotent closure operators on it sending arrows to the identity arrows at their source and target, and composition is only defined where the images of these operators overlap (fg only exists if domain(f)=codomain(g), and then domain(fg)=domain(g) and codomain(fg)=codomain(f)). You can always recover objects from the arrow-only presentation, but technically they are just a convenient shorthand for static processes, where a "process" is just an arrow.

This makes sense to me, that objects can be made redundant, because they are simply places for arrows to go. And especially since (as I understand it) to be considered a category, each object needs an identity arrow, such that an object could be considered just this arrow, thereby making the object reduntant, yeah. And then composition would look, as you say, to be just defined where arrows align tip-to-tail.
